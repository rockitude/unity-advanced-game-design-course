The purpose of this course is to introduce (or re-introduce) students to games in multiple types of media, and to the video game industry. During the course, students will learn what makes a game a game while they try their hand at creating a game of their own. They will begin by creating a pitch that encapsulates their game idea in a document. They’ll then use that document as a guide while they create their own 3D video game.

Goals

By the end of the course, students will be able to:

  • Explain what a game is
  • Evaluate a game’s strengths and weaknesses
  • Describe the advantages and disadvantages of different types of game media (board, card, athletic, computer, etc.)
  • Describe the platform and genre that fit a given game
  • Differentiate between common game genres
  • Describe several of the professions that are involved in creating a video game
  • Create a complete pitch for a new game of the student’s own design
  • Create a vertical slice (small, complete section) of a playable 3D video game that includes the following prefabricated or original assets:
    • Terrain
    • Props
    • Player and Non-Player Characters
    • Skybox
    • Cameras
    • Lighting
  • Apply textures to visual elements within a game
  • Apply custom scripts to manage user input for a game

Outline

Availabile Quest Chains

  • Press Start() - Starting quests that introduce students to the course and to 3D GameLab
  • Game Designer - This quest chain introduces to students to the fundamentals of game design. Students will learn how to break games down into their component parts, differentiate between game genres and platforms and complete game design projects.
  • Programmer - This quest chain uses Codecademy.com, an online resource for learning programming, to introduce students to the basics of the JavaScript scripting language, which they will be using to automate functions in Unity.
  • Object Oriented - This quest chain covers the basics of using Unity to design games. Students will learn important terminology and explore the development environment.
  • Watcher on the Wall - This quest chain covers the use of cameras.
  • In Control - This quest chain covers components related to character control and input.
  • Triggerer - This quest chain covers the use of triggers and colliders.
  • Invaluable Asset - This quest chain explains how to find, create and use outside assets in your game.
  • Light in the Darkness - This quest chain covers the use of lights.
  • What’s the Mats? - This quest chain covers materials and textures.
  • Game Controller - This quest chain covers using scripting to manage user input and automate actions in your game.

Course Objectives

Chain 1: What is a Game?

Students will be able to:

  • Create games that exemplify the traits that are common to all games.
  • Build, experiment and iterate with prototypes to test and improve their game designs.
  • Develop clear, concise and comprehensive instruction sets.

Chain 2: Genres, Platforms and Elements of Modern Games

Students will be able to:

  • Differentiate between game platforms and identify their strengths and weaknesses.
  • Differentiate between game genres and identify each one’s unique attributes.
  • Analyze game ideas and proposed features to identify platforms that fit a proposed design.
  • Create comprehensive technical documentation and planning materials for their game.

Chain 3: Introduction to Unity: Navigating the Interface

Students will be able to:

  • Apply game development terminology and concepts in their projects.
  • Relate concepts of inheritance in computer science to examples in the real world.
  • Utilize Unity’s features and interface to create a static game scene.
  • Manipulate objects in a scene using Unity’s toolset.

Chain 4: Adding and Manipulating Game Objects

Students will be able to:

  • Relate Unity’s Components view to object properties in computer science and real-world examples.
  • Create in-game objects at runtime using prefabs.
  • Participate in the Unity asset economy by importing and exporting assets from their projects.
  • Write syntactically and logically correct computer code.

Chain 5: Lighting, Texturing and Cameras

Students will be able to:

  • Build realistic game objects using surface textures, materials and shaders.
  • Choose from a variety of lighting objects and settings to fit a game state.
  • Create, select and apply texture and skybox materials to add realism to their game.
  • Apply colliders and triggers to create realistic object behaviors.

Chain 6: User Input

Students will be able to:

  • Accept keyboard and mouse input from a player.
  • Create automated object behaviors in their game.
  • Create repeatable and customizable scripts.
  • Demonstrate proper implementation of important computer science concepts.
